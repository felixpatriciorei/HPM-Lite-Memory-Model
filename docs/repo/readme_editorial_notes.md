# README editorial notes

This README is written as a GitHub front page, not as a technical report.

Design choices:

- First screen says what the project is, why it exists, and what result to look at.
- Main result appears before architecture details.
- One primary figure is shown near the top.
- Claim boundaries are kept, but moved into a scope/limitations section instead of repeated throughout the introduction.
- Full methodology remains in docs and processed CSVs; the README points to them instead of reproducing everything.
- All current figure links use `results/figures/research_grade/`.
- Old `paper`, `advanced`, and `docs/figures` links are not used.

External references used for style direction:

- GitHub's README guidance: explain why the project is useful, what users can do with it, and how to use it.
- Write the Docs: write for readers, keep documentation usable, and avoid unnecessary complexity.
- The Turing Way: keep reproducible research projects transparent and easy to rebuild.

This file is just process documentation. The README itself should stay clean.
